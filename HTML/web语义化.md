# web 语义化

> web 语义化是指`使用恰当的 HTML 标签和 class 类名等内容`，让页面具有良好的结构和含义，从而让人和机器都能快速理解网页内容。

## web 语义化优点

1. 正确的标签做正确的事情
2. 页面内容结构化
3. 无 css 样式时也容易阅读，便于阅读维护和理解
4. 便于浏览器、搜索引擎解析。便于爬虫标记、利于网站 SEO

## HTML 语义化

> HTML 为网页文档内容提供上下文结构和含义。

> 对于 HTML 语义化是指，使用语义恰当的标签，使页面有良好的结构，让页面元素有含义，便于被浏览器、搜索引擎解析、利于 SEO。

### HTML 语义化标签

1. body
2. article

> article 代表一个在文档，页面或者网站中自成一体的内容，其目的是为了让开发者独立开发或重用

> 注意

    1. 自身独立的情况下：用 article
    2. 是相关内容：用 article
    3. 没有语义的：用 div

3. nav

> nav 代表页面的导航链接区域。用于定义页面的主要导航部分

> 注意

    1. 用于整个页面的主要导航部分，不合适就不要使用 nav 元素

4. aside

> aside 被包含在 article 元素中作为主要内容的附属信息部分，其中的内容可以是与当前文章有关的相关资料，标签和名词解释

> aside 在 article 之外使用作为页面或站点全局的附属信息部分。最典型的是侧边栏。

> 注意

    1. aside 在 article 内表示主要内容的附属信息
    2. 在 article 之外可以作为侧边栏，没有 article 与之对应，最好不用
    3. 如果是广告，其他日志链接或者其他分类导航也可以用

5. section

> section 代表文档中的"节"或"段"。"节"可以是指一个页面中的分组。"段"可以是指一篇文章中按照主题的分段

> 注意

    1. 一张页面中可以用 section 划分为简介、文章条目和联系信息等内容。不过在文章内页，最好用 article。section 不是一般意义上的容器元素，如果想作为样式展示和脚本便利可以用 div
    2. 表示文档中的节或者段

6. header

> header 代表 "网页"或"section"的页眉，通常包含 h1-h6 元素或者 hgroup，作为整个页面或或者一个内容块的标题。

> 注意

    1. 可以是"网页"或者任意"section"的头部部分
    2. 没有个数限制
    3. 如果 hgroup 或者 h1-h6 自己能工作的很好，那么就没必要使用header

7. footer

> footer 代表"网页"或者"section"的页脚，通常含有该节的一些基本信息，譬如：作者、相关文档链接、版权资料。如果 footer 元素包含了整个节，那么它们就代表附录、索引、许可协议等信息

> 注意

    1. 可以是"网页"或者任意"section"部分的底部部分
    2. 没有个数限制，除了包裹的内容不一样，其他跟header类似

8. hgroup

> hgroup 代表"网页"或"section"的标题，当元素有多个层级时，该元素可以将 h1-h6 元素放在其中。

> 注意

    1. 如果只需要一个 h1-h6 标签就不用 hgroup
    2. 如果有连续的 h1-h6 标签就用 hgroup
    3. 如果有连续多个标题和其他文章数据， h1-h6 标签用 hgroup 包住，和其他文章元数据一起放在 header 标签

9. ......

### HTML 语义化总结

> HTML 语义化是反对大篇幅使用无语义的 `div+span+class`，而是鼓励使用 HTML 语义好的语义化标签

1. 使用 DOM 操作来添加这些标签
2. 使用封装好的库 html5shlv.js
